# Security Policy

OpenLLM Audit is intended for authorized testing of local LLM systems.

Please use it only on systems you own, operate, or have permission to test.

## Reporting Security Issues

If you find a security issue in OpenLLM Audit, please do not open a public issue with exploit details.

Contact:

- Please use the repository security contact or open a private security advisory if available.

Please include:

- a short description of the issue
- affected version or commit
- steps to reproduce, if safe to share
- any logs or output that help explain the problem

If testing reveals a security issue in another open-source tool, please report it to that project's maintainers through their preferred security process.
